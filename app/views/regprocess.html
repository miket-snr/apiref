How the Login Works

Step 1:   User Registration

The User has to be registered with an Authentication Server. For this API demo, the Authentication Server is SAP and the User MUST be registered as a Business Partner

For the demo just use any email address and the password "newP0ss" . No data can be changed with this logon and in time it will be removed

Step 2:   Token Issued

On successful Authentication, the User will be issued a TOKEN that is a very long set of alphanumeric characters by the OAuth process in the API. This is a simple call to https://... .... /Token and note the SSL requirement. In the body of the call there has to be three parameters:

grant_type: 'password'

user: user-email entered

password: user password entered

This call will return a long string that is known as a "token" and must be used for later calls - it is useable by the User on any device and will expire in 30 minutes (we can change this by design ) But the Token has to be kept secure - hence all calls are https//...

Step 3:   Token Usage

The API Service is exposed to all and sundry on the Internet and has to be secured and so the API has to be in a certain format with an ISSUED TOKEN and using the https:// Protocol.

There is a lot of information available on Google or Bing regarding the usage of TOKENS.
